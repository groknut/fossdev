

# Make
Make выступает как единая точка входа для управления окружением, зависимостями и проверками качества кода. Каждое задание должно не просто требовать добавить новый target, но и заставлять задуматься о том, как взаимодействуют Bash-команды, Python-инструменты и переменные Make. Цель здесь — не выучить команды, а понять, как упаковать хорошие практики разработки в воспроизводимое действие.

## Что нужно сдать:
- Исходный код (код, документация, конфиги и пр.)
- Автоматизацию действий (После прохождения Mйake).
- Документация (README) в формате Markdown в папке проекта, подробно описывающий предложенное решение. 

## Общие требования
- После проверки задания  (запуска, сборки, тестирования) в репозитории git не должен показывать новые файлы (gitignore должен быть настроен верно)
- Работа в команде (независимо от того работаете ли вы в одном репозитории или в форке): для каждого задания нужно создать ветку в которую вы потом сольете работку каждого из участников (не забудьте разрешить конфликты)
- By-design not by-mistake:  должно быть явно демонстрироваться, что вы сделали, то что задумали, а не потому что повезло (например если мы ставим библиотеку, она точно ставиться в  окружение проекта, а не в системное окружение).

## Работа с окружением (stateless)
**Makefile:** [link](./stateless/Makefile)
```bash
make venv # создать виртуальное окружение
make install # установить
make run # запустить src/app.py
make clean # очистить
```

## Проверка зависимостей. 
**Makefile:** [link](./check_req/Makefile)
```bash
# python ./src/check_req.py  requirements.txt ./src/service.py
make check-requirements
```

## Проверка типов.
**Makefile:** [link](./check_type/Makefile) \
`$(VENV)/.done` используется для того, чтобы не пересоздавать виртуальное окружение каждый раз
```bash
make install # устанавливаем виртуальное окружение
make type-check # проверяем ошибки типов с mypy
```

## Проверка стилей. 
**Makefile:** [link](./styles/Makefile) \
Для проверки стилей используется `ruff`
```bash
make install # устанавливаем ruff
make format # проверка на неконсистентное форматирование
make lint # замена неконсистентного форматирования
```
